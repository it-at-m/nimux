2. Description of the Application

NIMux is a native Android application designed to centrally manage supplies, inventory, materials and operational resources. This privacy policy is necessary for the published version of NIMux in the Google play store.

The application allows authorized users to:

  • manage inventory items
  • track stock levels
  • record check-outs of materials
  • organize operational resources

The project also demonstrates the capabilities of Android applications using Firebase services.

3. Restricted Access

The application is not intended for public use.

Access is restricted to a defined internal group of authorized users.

Users can only access the application if an account has been created for them by the system administrator.

External users cannot register themselves. Unauthorized login attempts are automatically rejected by Firebase Authentication.

4. Authentication (Firebase Authentication)

User authentication is handled using Firebase Authentication, a service provided by Google Ireland Limited, Gordon House, Barrow Street, Dublin 4, Ireland. During authentication attempts, personal data such as email address, IP address, login timestamps and technical device information may be processed. This processing is required to perform authentication, ensure system security and prevent unauthorized access. Login attempts by users without authorization are automatically rejected by the authentication system. Authentication data is stored only as long as the corresponding user account exists. When an account is deleted, the associated authentication data is removed.

5. Database (Firebase Firestore)

The application uses Firebase Firestore as a cloud database. Application data is stored using Google Cloud infrastructure. The database is hosted in the europe-west3 (Frankfurt, Germany) region and therefore located within the European Union.

Stored data may include:

  • inventory items
  • materials and resources
  • inventory transactions
  • organizational units
  • user roles and permissions

This data is necessary to provide the functionality of the application.

6. Internal Application Users

Administrators may create internal user profiles within the application database.

These users:

  • are not Firebase Authentication accounts
  • exist only for internal documentation purposes
  • cannot log into the application

Stored information may include:

  • name or identifier
  • internal ID
  • associated transactions

7. Legal Basis

Data processing is based on Article 6(1)(f) GDPR (legitimate interests).

The legitimate interest consists of:

  • inventory management
  • tracking resource usage
  • organizational accountability

8. Data Sharing

Personal data is not shared with third parties except technical service providers required for operating the application.

This includes:

  • Google Firebase (authentication and database services)

Personal data is not sold or used for advertising purposes.

9. Data Retention

Data is stored only as long as necessary for the operation of the application.

Data may be deleted when:

  • a user leaves the organization
  • access is revoked
  • the application is discontinued
